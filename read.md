О задаче
Пользователи Авито обращаются в поддержку с короткими вопросами:

как отправить товар покупателю,
почему объявление всё ещё на проверке,
как оформить возврат,
где посмотреть статус заказа,
что делать, если не проходит оплата.
В продуктовой системе такие статьи не используются как готовый ответ пользователю. Они служат документами для RAG-пайплайна: сначала нужно найти подходящие материалы, а уже потом на их основе формируется ответ.

В этой задаче нужно решить этап поиска: по тексту вопроса пользователя найти статьи, которые стоит передать дальше в пайплайн.

Цель задания
Разработать решение, которое:

Принимает текст пользовательского вопроса.
Ищет релевантные документы в базе статей справочного центра.
Возвращает ранжированный список article_id: сначала самые подходящие документы, затем менее подходящие.
В одном запросе может быть несколько правильных статей. Поэтому важно не только найти одну наиболее похожую статью, но и расположить все релевантные документы как можно выше в топ-10.Входные данные
Тебе доступны три файла в формате Feather:

articles.f — база статей справки.

Колонки:

article_id — целочисленный идентификатор статьи.
title — заголовок статьи.
body — текст статьи в HTML.

Текст статей дан в том виде, в котором он хранится в базе: с HTML-разметкой, таблицами, вкладками, спойлерами и другими служебными элементами.

calibration.f — 500 размеченных запросов для локальной проверки решения.

Колонки:
query_id — идентификатор запроса.
query_text — текст вопроса пользователя.
ground_truth — правильные article_id, разделённые пробелами.

Пример:

query_text: "Здравствуйте,подскажите как мне отправить кроссовки покупателю через пункт «Авито»?"

ground_truth: "1909 4396"

Это значит, что для запроса в RAG-пайплайн нужно передать статьи 1909 и 4396.

test.f — 500 запросов без разметки. Для них нужно подготовить ответ.

Колонки:

query_id — идентификатор запроса.
query_text — текст вопроса пользователя.

Файлы можно скачать на платформе через кнопку "Скачать набор данных".

Ожидаемый результат
Ответ нужно подготовить в CSV-файле answer.csv

В файле должны быть ровно две колонки:

query_id — тот же идентификатор, что в test.f
answer — строка с ранжированным списком article_id, разделённых пробелами.

В answer.csv должны быть ответы для всех query_id из test.f: без пропусков, лишних строк и повторов. Для каждого запроса можно вернуть не больше 10 статей. Повторять один и тот же article_id внутри одного ответа нельзя.
Все article_id должны существовать в articles.f.

Нужно возвращать именно идентификаторы статей, а не ссылки, названия статей или идентификаторы разделов.

Если у решения есть ранжированный список кандидатов, обычно имеет смысл возвращать до 10 статей: дополнительные документы в конце списка не ухудшают метрику, если порядок уже выбранных документов не меняется.Формат ответа
Пример answer.csv:

query_id,answer
1,1909 4396 4403 2693 4433 2408 4219 4218 4217 4216
2,1730 1746 1747 1774 1775 1776 1784 1809 1824 1825
3,1827 1828 1829 1830 1831 1833 1834 1835 1836 1837

                  
Порядок важен: документ на первой позиции считается более релевантным, чем документ на второй.

Сохранение предсказаний в Python
import pandas as pd

source_data = pd.read_feather("test.f")

                  
Здесь должен быть твой код ранжирования.

Функция your_solution должна вернуть строку с article_id через пробел.

source_data["answer"] = source_data["query_text"].apply(your_solution)

source_data[["query_id", "answer"]].to_csv("answer.csv", index=False)

                  
Целевая метрика
Решение будет оцениваться по метрике MAP@10.

Для одного запроса считается AP@10: чем выше в списке стоят релевантные документы, тем больше их вклад. Затем значение усредняется по всем запросам.

Если для запроса есть несколько правильных статей, в оценке учитываются все найденные документы, а не только первый.

Формально для запроса q с множеством правильных статей R_q и вашим ранжированным списком p_1, ..., p_k, где k <= 10:

AP@10 = (1 / min(|R_q|, 10)) * sum_i [p_i in R_q] * Precision@i

MAP@10 — среднее значение AP@10 по всем запросам.

У тебя будет 7 попыток. Засчитывается наилучший результат.
